linkedDoc

  • linkedDoc(options: object): Doc
    • Defined in typings/codemirror/codemirror.d.ts:491

    Create a new document that's linked to the target document. Linked documents will stay in sync (changes to one are also applied to the other) until unlinked.

    Parameters

    • options: object
      • Optional from?: number
      • mode: any

        By default, the new document inherits the mode of the parent. This option can be set to a mode spec to give it a different mode.

      • Optional sharedHist?: boolean

        When turned on, the linked copy will share an undo history with the original. Thus, something done in one of the two can be undone in the other, and vice versa.

      • Optional to?: number

        Can be given to make the new document a subview of the original. Subviews only show a given range of lines. Note that line coordinates inside the subview will be consistent with those of the parent, so that for example a subview starting at line 10 will refer to its first line as line 10, not 0.

    Returns Doc

setBookmark

  • setBookmark(pos: Position, options?: object): TextMarker
    • Defined in typings/codemirror/codemirror.d.ts:538

    Inserts a bookmark, a handle that follows the text around it as it is being edited, at the given position. A bookmark has two methods find() and clear(). The first returns the current position of the bookmark, if it is still in the document, and the second explicitly removes the bookmark.

    Parameters

    • pos: Position
    • Optional options: object
      • Optional insertLeft?: boolean

        By default, text typed when the cursor is on top of the bookmark will end up to the right of the bookmark. Set this option to true to make it go to the left instead.

      • Optional widget?: HTMLElement

        Can be used to display a DOM node at the current location of the bookmark (analogous to the replacedWith option to markText).

    Returns TextMarker
