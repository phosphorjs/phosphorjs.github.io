Class Component<T>

A concrete implementation of IComponent with virtual DOM rendering.

User code should subclass this class to create a custom component. The subclasses should reimplement the render method to generate the virtual DOM content for the component.

Protected shouldUpdate

  • shouldUpdate(data: T, children: IElement[]): boolean
    • Defined in src/components/component.ts:126

    Test whether the component should be updated.

    This method is invoked when the component is initialized with new data and children. It should return true if the component should be updated, or false if the values do not cause a visual change.

    Determining whether a component should update is error prone and can be just as expensive as performing the virtual DOM diff, so this should only be reimplemented if performance is a problem.

    The default implementation of this method always returns true.

    Parameters

    • data: T
    • children: IElement[]

    Returns boolean

update

  • update(immediate?: boolean): void
    • Defined in src/components/component.ts:87

    Schedule a rendering update for the component.

    This should be called whenever the internal state of the component has changed such that it requires the component to be re-rendered, or when external code requires the component to be refreshed.

    If the 'immediate' flag is false (the default) the update will be scheduled for the next cycle of the event loop. If the flag is set to true, the component will be updated immediately.

    Multiple pending requests are collapsed into a single update.

    Parameters

    • Default value immediate: boolean = false

    Returns void
